# <a name="workflows-in-sharepoint"></a>SharePoint 中的工作流

<p>SharePoint 工作流是预先设计的小型应用程序, 可简化和自动执行各种业务流程。 工作流的范围可以从收集计划或文档的签名、反馈或审批中获取, 以跟踪常规过程的当前状态。</p> <p>入门<strong>:</strong>&rsquo;如果只是开始 sharepoint 工作流, 请参阅<a href="https://support.office.com/en-us/article/introduction-to-sharepoint-workflow-07982276-54e8-4e17-8699-5056eff4d9e3">Sharepoint 工作流简介</a>了解详细信息。</p> <p><strong>Microsoft Flow:</strong>考虑使用 Microsoft Flow 而不是经典 SharePoint 工作流解决方案。 有关详细信息, 请参阅<a href="https://support.office.com/en-us/article/create-a-flow-for-a-list-or-library-in-sharepoint-online-or-onedrive-for-business-a9c3e03b-0654-46af-a254-20252e580d01">在 SharePoint Online 或 OneDrive 中创建列表或库的流</a>。</p> <p><strong>工作流疑难解答和调试:</strong>调试工作流类似于早期版本的工作流。 有关详细信息, 请参阅<a href="https://docs.microsoft.com/en-us/sharepoint/dev/general-development/debugging-sharepoint-server-workflows">调试 SharePoint 工作流</a>。 有关故障排除的详细信息, <a title="请参阅 SharePoint Server 工作流验证错误疑难解答" href="https://docs.microsoft.com/en-us/sharepoint/dev/general-development/troubleshooting-sharepoint-server-workflow-validation-errors-in-visio">解决 SharePoint Server 工作流验证</a>错误。&nbsp;</p>
