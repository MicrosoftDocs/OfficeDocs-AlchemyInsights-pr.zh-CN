1. <span data-ttu-id="ec192-104">立即[重置用户密码](https://support.office.com/article/7a5d073b-7fae-4aa5-8f96-9ecd041aba9c)。</span><span class="sxs-lookup"><span data-stu-id="ec192-104">[Reset the user's password](https://support.office.com/article/7a5d073b-7fae-4aa5-8f96-9ecd041aba9c) immediately.</span></span> <span data-ttu-id="ec192-105">请勿通过电子邮件将新密码告知最终用户。</span><span class="sxs-lookup"><span data-stu-id="ec192-105">Do not communicate the new password through email to the end user.</span></span>

2. <span data-ttu-id="ec192-106">删除邮箱级别的所有可疑[转发地址](https://support.office.com/article/ab5eb117-0f22-4fa7-a662-3a6bdb0add74)。</span><span class="sxs-lookup"><span data-stu-id="ec192-106">Remove any suspicious [forwarding addresses](https://support.office.com/article/ab5eb117-0f22-4fa7-a662-3a6bdb0add74) set at the mailbox level.</span></span>

3. <span data-ttu-id="ec192-107">删除邮箱中设置的所有可疑[收件箱规则](https://support.office.com/article/1433E3A0-7FB0-4999-B536-50E05CB67FED)。</span><span class="sxs-lookup"><span data-stu-id="ec192-107">Remove any suspicious [inbox rules](https://support.office.com/article/1433E3A0-7FB0-4999-B536-50E05CB67FED) set within the mailbox.</span></span>

4. <span data-ttu-id="ec192-108">如果用户被阻止发送电子邮件，[找到受限制的用户解锁帐户](https://protection.office.com/?hash=/restrictedusers)。</span><span class="sxs-lookup"><span data-stu-id="ec192-108">If the user is blocked from sending email, [go to the Restricted Users to unblock the account](https://protection.office.com/?hash=/restrictedusers).</span></span> <span data-ttu-id="ec192-109">完成后，用户应该可以在在 1 小时内继续发送邮件。</span><span class="sxs-lookup"><span data-stu-id="ec192-109">Once done, the user should be able to resume sending messages within 1 hour.</span></span>

5. <span data-ttu-id="ec192-110">从任何[管理角色组](https://support.office.com/article/eac4d046-1afd-4f1a-85fc-8219c79e1504)删除用户帐户，直到确定帐户不再被盗用。</span><span class="sxs-lookup"><span data-stu-id="ec192-110">Remove the user account from any [administrative role groups](https://support.office.com/article/eac4d046-1afd-4f1a-85fc-8219c79e1504) until you are confident that the account is no longer compromised.</span></span>
