# <a name="sharepoint-alert-notifications-not-delivered"></a>未传递 SharePoint 通知通知 

确定是否未传递所有警报, 或者是否未传递来自特定文件或库的单个警报。

如果未传递来自多个文件或库的所有警报, 请访问[服务运行状况仪表板](https://admin.microsoft.com/AdminPortal/Home#/servicehealth), 以检查任何可能与 SharePoint 或 Exchange 一起发生的咨询/事件。 问题可能与 SharePoint 通知功能或通过 Exchange 的电子邮件延迟有关。 此外, 请务必注意是否正在传递其他电子邮件, 如果没有, 则可能是 Exchange 延迟问题。 

如果未传递来自特定文件或库的单个通知, 则可以尝试删除并重新创建, 请参阅[管理、查看或删除 SharePoint 警报</a>以重新创建通知](https://support.office.com/article/manage-view-or-delete-sharepoint-alerts-99dfb19c-9a90-4a8c-aba1-aa8c8afb0de2#ID0EAADAAA=Online)。 

有关设置通知的详细信息, 请参阅[创建通知以在 SharePoint 中的文件或文件夹更改时获得通知](https://support.office.com/article/create-an-alert-to-get-notified-when-a-file-or-folder-changes-in-sharepoint-e5a79e7b-a146-46da-a9ef-d65409ba8918)。

有关排除警报的详细信息, 请参阅[用户不接收 SharePoint Online 警报通知](https://support.office.com/article/users-don-t-receive-sharepoint-online-alert-notifications-14fc22dd-e873-482c-844d-f67ad41313f1)。
